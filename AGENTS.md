# 仓库协作指令

1. 必须优先使用 Bun 运行、构建和测试；如果环境里没有 Bun，需要先安装再继续。
2. 必须实现用户要求的完整功能，不能只做最小可运行版本或只留下占位内容。
3. 在宣告完成之前，必须实际运行项目并提供界面截图，确认改动已经生效。
4. 产品面向不懂 IT 的普通用户，界面文案要尽量使用自然中文，避免技术术语，避免英文。
5. 不能依赖 Node 作为运行前提，所有脚本、构建和测试都要能直接用 Bun 执行，并保持 `bun build --compile` 打成单文件可运行。
6. 预览和渲染功能必须统一基于 `@matbee/libreoffice-converter` 实现，不允许引入其他渲染实现或并行维护第二套出图链路。
7. 所有任务都需要创建 PR。

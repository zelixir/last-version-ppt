## 使用说明

首先你需要有一个兼容openai协议的大模型服务, 如openai/阿里云等
配置完成后输入需求并点击创建项目, 然后AI就会开始制作PPT
此期间你可以在网页上预览PPT, 或者上传一些图片让AI设置为封面
也可以上传一些文本文件(或者csv表格), 用于制作具有真实内容的PPT
针对不满意的部分就通过对话让AI修改
如果你的模型支持多模态, AI可以直接读取生成后的ppt并检查效果
还有其他不懂的, 也可以直接问AI

## 网页预览PPT是怎么实现的

一开始没想到在网页上预览ppt是传说级任务
让AI来做, 结果AI欺上瞒下整了个最小实现, 等我发现的时候天都塌了
渲染PPT文件的功能基本上只有一些商业的库才支持, js是不可能支持的
不过好在有wasm, 理论上可以将c#的库编译成wasm放在前端或者后端环境上调用
只要找到一个开源的能渲染PPT的c#库就行了
不过实际上也没这么简单, 这么庞大的库总会依赖一些奇奇怪怪的环境, 反正到了wasm就是跑不起来
发动人海战术, 几个旗舰模型全部开起来让它们找方案, 然后找到了@matbee/libreoffice-converter这个库
一开始想着在前端运行, 结果这个库经常会卡死, 后来尝试放到后端, 又发现不兼容bun
经过漫长的尝试, 最终让AI修改了@matbee/libreoffice-converter原始项目, 让它兼容了bun

## 成本
这个项目亏麻了!!

主要是网页预览PPT功能, 花了10天左右, 几个模型并行跑, 每次都跑够一个小时超时
因为每次都要跑好久, 所以一天就开两三次任务, AI额度还没超, action额度先超了
烧了十几块钱action额度之后开了个小号来跑, 大概加起来就几十块钱的服务吧
一共开了72次agent, 按平均一次消耗15分钟人力, 算下来是18小时
